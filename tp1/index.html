Exercice 1 : Mémo Flexbox (Cheat Sheet)

Mon analyse

Connaissances : Ici, l'objectif n'était pas de construire une page web classique, mais de structurer de la donnée technique. J'ai dû organiser visuellement les propriétés du module Flexbox en distinguant clairement le conteneur parent des éléments enfants via des tableaux HTML.
Difficulté : Moins complexe techniquement que de la logique pure, la difficulté résidait dans la rigueur de l'intégration. Il fallait gérer la densité d'informations textuelles sans surcharger la page, tout en gardant un code HTML sémantique et propre.
Intérêt : C'est sans doute l'exercice le plus "rentable" à long terme. Plutôt que de coder une page jetable, j'ai créé mon propre outil de référence. Cela m'a forcé à visualiser concrètement la différence fondamentale entre les axes (Main vs Cross) que je confondais souvent.
Qualité : Le résultat est un document technique fonctionnel. L'utilisation de classes utilitaires pour les couleurs (vert/rouge) rend la lecture instantanée et facilite la mémorisation des concepts.

Exercice 2 : Mise en page contrainte (CSS seul)

Mon analyse

Connaissances : L'épreuve ici était de reproduire une maquette sans toucher une seule ligne de HTML. J'ai dû ruser avec le CSS, notamment en utilisant les propriétés background pour placer les images de décoration et le positionnement relatif/absolu.
Difficulté : C'est le type d'exercice que je trouve le plus "scolaire" et fastidieux car il exige une précision extrême. J'ai rencontré des difficultés pour aligner parfaitement le texte sans pouvoir modifier la structure de la page.
Intérêt : Malgré la contrainte, cela m'a forcé à mieux comprendre le flux du document et à réaliser à quel point le CSS peut transformer radicalement l'apparence d'un contenu brut.
Qualité : Le résultat respecte la maquette imposée, prouvant qu'on peut séparer totalement le fond (HTML) de la forme (CSS).

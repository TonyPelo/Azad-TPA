Exercice 2 : Comparatif Flex vs Grid

L'objectif était de reproduire la même mise en page avec trois techniques différentes pour en comprendre les nuances.

1. Grid Classique

2. Grid Template Areas

3. Flexbox

Mon analyse

Connaissances : Cet exercice m'a permis de manipuler grid-template-columns (Grid Classique), grid-template-areas (Grid Area) et display: flex imbriqués.
Difficulté : La version Flexbox (ex2.3) était la plus complexe à structurer car elle nécessitait d'ajouter des divs HTML supplémentaires pour gérer les alignements 2D. À l'inverse, Grid Area (ex2.2) était la plus intuitive.
Intérêt : Comparer ces méthodes prouve que CSS Grid est bien plus adapté que Flexbox pour définir l'architecture globale d'une page (Header, Sidebar, Content, Footer).
Qualité : Le rendu visuel est identique, mais le code de la version "Grid Area" est le plus propre et le plus facile à maintenir.

Exercice 3 : Puzzle Grid

Mon analyse

Connaissances : J'ai utilisé grid-template-areas pour placer précisément 9 éléments dans une grille complexe, combiné à Flexbox pour le centrage interne.
Difficulté : Niveau moyen. Il fallait réussir à traduire le schéma visuel du puzzle en une syntaxe de grille CSS (chaînes de caractères).
Intérêt : Cet exercice apprend à maîtriser le positionnement explicite. On ne dépend plus de l'ordre HTML, on place les éléments visuellement via le CSS.
Qualité : L'utilisation de aspect-ratio: 1 et border-radius: 50% garantit des formes géométriques parfaites sur tous les écrans.

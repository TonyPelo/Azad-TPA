Exercice 1 :Media Queries

Mon analyse

Connaissances : J'ai mis en pratique les Media Queries (`@media`). J'ai défini des points de rupture (breakpoints) à 960px et 480px pour modifier dynamiquement les propriétés CSS (ici la couleur de fond) selon la largeur de l'écran.
Difficulté : Niveau débutant. Le concept est simple : une condition de largeur déclenche un style différent.
Intérêt : C'est le fondement technique absolu du Responsive Web Design. Sans Media Queries, il n'y a pas d'adaptation mobile possible.
Qualité : Le résultat est immédiat : au redimensionnement de la fenêtre, le bloc réagit instantanément, prouvant que la feuille de style conditionnelle fonctionne.

Exercice 3 : Menu Burger

Mon analyse

Connaissances : J'ai transformé un menu horizontal classique (Flexbox) en un menu vertical "Burger" pour mobile. J'ai utilisé une astuce avancée avec le pseudo-sélecteur CSS :target pour gérer l'ouverture et la fermeture du menu sans utiliser une seule ligne de JavaScript.
Difficulté : Niveau intermédiaire. Gérer le positionnement absolu du bouton burger, le masquage du menu (`max-height: 0`) et l'animation d'ouverture demande de la précision.
Intérêt : Le "Menu Burger" est le standard de navigation sur mobile. Savoir le réaliser en CSS pur permet de créer des interfaces légères et performantes.
Qualité : Le menu est fluide et l'animation de déroulement (`transition`) rend l'expérience utilisateur agréable.

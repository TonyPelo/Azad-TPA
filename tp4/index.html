Exercice 1 :

Mon analyse

Connaissances : Dans cet exercice, j'ai créé un système de navigation par onglets sans utiliser de JavaScript. J'ai utilisé position: fixed pour figer le menu à gauche, et surtout la pseudo-classe CSS :target.
Technique : Par défaut, les paragraphes sont cachés (`display: none`). Lorsqu'on clique sur un lien, l'URL change (ex: #paragraphe1), ce qui active le CSS div:target p { display: block; } sur l'élément ciblé.
Intérêt : Cela montre qu'on peut créer de l'interactivité (afficher/masquer du contenu) uniquement avec du CSS, ce qui est très performant.

Exercice 2 :

Mon analyse

Connaissances : J'ai utilisé les sélecteurs d'attributs avancés (ex: [href$=".pdf"]) pour cibler les liens en fonction de l'extension du fichier pointé. J'ai combiné cela avec les pseudo-éléments ::before et ::after pour injecter du contenu (texte et icônes) automatiquement via CSS.
Difficulté : Niveau intermédiaire. Il faut comprendre que le HTML est volontairement vide et que c'est le CSS qui "remplit" les liens en lisant leurs attributs `href` et `hreflang`.
Intérêt : C'est une technique puissante pour l'automatisation et la maintenance. Si je change un fichier PDF par un Word, l'icône et le texte changent tout seuls sans que je modifie le texte du lien.
Qualité : Le code est propre et sémantique. Les liens sont standardisés et l'utilisateur a un repère visuel immédiat sur le type de document qu'il va télécharger.
